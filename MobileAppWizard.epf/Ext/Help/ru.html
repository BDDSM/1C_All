Данная обработка предназначена для формирования файлов поставки мобильного приложения. С ее помощью можно сформировать загружаемый файл для ОС Android и файл проекта для ОС iOS. Сформированный файл проекта необходимо использовать для построения приложения с помощью системы XCode.

Мобильное приложение можно формировать отдельно для ОС Android и отдельно для ОС iOS. За это отвечают соответствующие флажки на форме.

Для мобильного приложения необходимо задать Идентификатор приложения - уникальное имя Java-класса, которое будет использоваться в дальнейшем для выполнения обновления. Рекомендуется начинать данное имя с суффикса com.e1c. Для мобильного приложения необходимо задать Представление приложения - текст, который будет выводится в качестве подписи к иконке мобильного приложения. Иконки и логотипы приложения можно переопределить с помощью команды Задать иконки и логотипы. Каждое мобильное приложение обладает Версией приложения и последовательным Номером собрки.

Мобильное приложение может состоять из одной или нескольких конфигураций. За переключение между режимами отвечает команда Использовать несколько конфигураций/Использовать одну конфигурацию.

Для того, чтобы помощник смог выполнить свою работу, его необходимо настроить. Для этого служит команда Настройка. Имеется возможность сохранить и восстановить настройки в специальный конфигурационный файл по умолчанию: MobileAppWizard.cfg. Кроме того, можно работать с любыми другими конфигурацинными файлами: сохранить такой файл можно с помощью команды Записать настройки как, а прочитать - с помощью команды Открыть файл настроек.

Нажатие на кнопку Сформировать начинает процесс формирования мобильных приложений для выбранных операционных систем. Если формирование будет невозможно - система выдаст диагностические сообщения о причине невозможности формирования. В случае успешного завершения процесса будет выдано сообщение о том, куда размещены сформированные файлы.
